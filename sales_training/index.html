In order to access DDi Inform, our corporate ERP System, you must connect to a trusted network. Our Virtual Private Network will allow you to access valuable resources on our server.


Please read the documentation (also located in the Blink document hub) for connecting to the VPN

Logging Into Server

After successfully connecting to the VPN, we need to download the DDI Application to the remote computer.
DDI Application for Windows
Download and extract the .zip file to the C:\Program Files (x86) folder. If you navigate to this folder, you should find the application under the \DDI SYSTEM\inForm\bin folder.
Right-click on the DDI inForm application file and "Add to Taskbar" for easy access.

There is still some additional configuration to be done.
Click on the DDI icon. Click on "Connection Update". Fill out the configuration screen:

The 'Windows User Password' is: /0626inform
At this point, you should be able to launch DDI Inform and connect using your credentials. If you have any problems, please contact support, via Blink.

Logging Into DDI Inform

Once logged into the server, launch Inform by double-clicking on the icon
You will be prompted with a login

Use the credentials assigned to you (case sensitive). Please only have one Inform session open at a time - we have a strict amount of licenses available.

DDi Inform

Before continuing on, please download and review the official DDI training documentation. The rest of the materials in this section will assume basic knowledge about the system and the sales modules.

Calendar

The use of the DDI calendar may be used for both personal and professional scheduling, but keep in mind that this is not a private calendar. Any user may view and/or change the events on any other calendar. Any event placed on this calendar should be considered public knowledge. For example, the event titled "Doctor's Appointment" is perfectly fine, whereas specific information about the nature of the appointment should be avoided.

Sales Order & Quote Entry

The process for Sales Order and Quote entry are nearly identical. When producing a quote for a customer, the system will apply a mark-up automatically (dependant on what category of customer it is). If you would like to quote a different price from the suggested, you should use a new feature that allows you to create a temporary "Price Contract":
By checking the "Price Contract" box and supplying an expiration date, the system will refer to this new price until the quote expires. This is useful because we don't want to neccesarily guarantee pricing for longer than our cost is guaranteed for. This is a great way to automate profit margin management. Both the "Ship Via" and "Ship Date" fields will automatically populate and do not need to be manipulated.
